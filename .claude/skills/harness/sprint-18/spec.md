# Feature Spec: Phase 4 -- Editing & Polish

## Sprint 0: Risk Consolidation (docs/RISKS.md)
**Goal**: Create a single risk register document consolidating all active residual risks from sprints 0-18, with status, owner/area tags, and resolution tracking.
**Verification Profile**: static
**Acceptance Criteria**:
1. File `docs/RISKS.md` exists with structured risk register (ID, description, status, owner/area, origin sprint, resolution note).
2. All 12 residual risks are present.
3. Resolved risks marked `resolved` with resolution note.
4. Each risk has a non-empty owner/area tag.
5. Document contains a "Resolution Log" section.
