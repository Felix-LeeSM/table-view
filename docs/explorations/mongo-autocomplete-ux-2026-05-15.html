Autocomplete UX 옵션 비교 (Mongo / SQL 공유)

현재 baseline 은 popup 외곽선 + 항목 highlight + ↑↓ navigate + Tab/Enter accept + Esc close. 아래 6 옵션은 그 위에 얹는 변형들. 모두 같은 popup component 에 적용되므로 SQL editor 의 SELECT/SELECTIVE popup 에도 동일하게 반영됨.

1 · 선택 highlight 강화 low cost

현재 active 항목은 보라색 배경뿐. 좌측에 3px accent bar 를 추가해 어두운 배경에서도 즉시 어디가 active 인지 인식.

1db.users.fi
ƒfind method
ƒfindOne method
ƒfindOneAndUpdate method

2 · 키 안내 hint bar low cost

popup 하단에 한 줄. 신규 사용자가 ↑↓/Enter/Esc 를 자동으로 학습. 폭이 늘어나 좁은 화면에서 거슬릴 가능성 트레이드오프.

1db.users.fi
ƒfind method
ƒfindOne method
ƒfindOneAndUpdate method
↑↓navigate ⏎accept Escclose

3 · Tab cycle (Vim/Helix 스타일) medium · 트레이드오프

Tab = next / Shift+Tab = prev / Enter = accept. fzf · Vim completion 친화. 단 RDB 워크플로우와 동시 변경되므로 사용자 hint 없이 도입하면 회귀 체감 가능.

1db.users.fi
ƒfind accept
ƒfindOne
ƒfindOneAndUpdate
Tabnext ⇧Tabprev ⏎accept

4 · 카테고리 시각 분리 medium

completion type 별로 아이콘 색을 분기 — 메서드/컬렉션/연산자/필드/ 타입태그 5종. 같은 prefix 라도 카테고리 즉시 식별. 색 5종은 한 popup 안에서만 쓰여 globally 노이즈는 적음.

5 · 함수 시그니처 inline medium

항목 옆에 `(filter, options?)` 형태로 호출 시그니처. mongosh 문서 참조 없이도 인자 순서 인지. detail 영역 폭이 늘어남.

1db.users.fi
ƒfind (filter?, options?)
ƒfindOne (filter, options?)
ƒfindOneAndUpdate (filter, update, options?)

6 · 우측 info 패널 higher cost

항목 선택 시 우측에 한 줄 설명 + 예제. CM 의 `info` API. 학습용으로 좋지만 popup 폭이 커지고 좁은 에디터에서 잘림 위험.

1db.users.find({ age: { $g } })
$$gt op
$$gte op
$$group stage

$gt

Matches values that are strictly greater than the operand. Example: { age: { $gt: 18 } }.
