진단

문서 체계 이탈

agent가 기존 SOT를 먼저 찾도록 강제되지 않았다. 그래서 audit 결과와 sprint sequencing을 RISKS, PLAN, sprint contract로 흡수하지 않고 새 임시 backlog 문서로 남겼다.

문서화 누락 가능성

어디에 써야 하는지만 정하면 부족하다. 중요한 변경이 문서화가 필요한지 판단하는 gate가 없으면, API, workflow, safety 정책 변경이 코드에만 남을 수 있다.

review와 merge 책임 분리 불명확

reviewer는 read-only 판단자여야 한다. merge는 delivery owner가 해야 한다. 현재는 reviewer 이후 reflect, fix, merge, cleanup을 소유하는 agent가 고정되지 않아 PR과 worktree가 쌓였다.

로컬 증거와 GPG 우회

PR에서 볼 수 없는 절대경로가 body에 들어갔고, GPG timeout에서 멈추지 않고 unsigned commit이 만들어졌다. 둘 다 프롬프트만으로는 취약하므로 reviewer gate와 hook 보강이 필요하다.

Documentation Impact Gate

문서화 필요 트리거

  • 사용자 가시 동작, default, shortcut, warning/confirm 정책 변경
  • IPC, store API, hook API, enum, SQL kind/severity 같은 contract 변경
  • agent, git, review, delivery, hook workflow 변경
  • password, signing, destructive command, safe mode 같은 safety 변경
  • CI, pre-push, test strategy, coverage threshold 변경
  • 새 invariant, deferred risk, follow-up이 생긴 경우

PR body 필수 섹션

모든 PR은 아래 판단을 포함한다. reviewer는 이 판단의 타당성을 본다.

  • Required: yes/no
  • Trigger: 무엇 때문에 문서화가 필요하거나 불필요한가
  • Updated SOT: 반영한 기존 문서
  • Not used: 새 임시 문서를 만들지 않은 이유

Review 관점

정합성

새 룰, 코드, 문서가 기존 ADR, memory, 실제 코드와 충돌하지 않는지 본다.

Scope

sprint contract의 in-scope/out-of-scope와 PR diff가 맞는지 본다.

Documentation topology

문서화가 필요한 변경인지, 필요한 경우 기존 SOT에 반영됐는지, 새 문서가 체계를 우회하지 않았는지 본다.

Evidence portability

PR body와 review comment에 GitHub에서 볼 수 없는 로컬 절대경로가 없는지 본다.

Lifecycle closure

review green 이후 merge, blocked report, worktree cleanup, agent close가 이어졌는지 본다.

Safety gate

GPG signing, no hook bypass, force push 금지 같은 delivery safety가 지켜졌는지 본다.

자동화 시나리오

1 Orchestrator task, scope, worktree path, delivery owner를 정의한다.
2 Delivery owner worktree에서 구현, commit, push, PR 생성을 소유한다.
3 PR body Documentation impact와 portable evidence를 포함한다.
4 Reviewer read-only로 정합성, scope, documentation topology를 평가한다.
5 Reflect finding이 있으면 같은 delivery owner가 수정한다.
6 CI Gate checks green, mergeable, policy unblocked인지 확인한다.
7 Merge delivery owner가 squash merge와 branch deletion을 수행한다.
8 Cleanup worktree cleanup, agent close, final report로 종료한다.

Agent Lifecycle

planned

목적, PR 번호, worktree, owner, 종료 조건이 등록된 상태.

running

worker나 reviewer가 작업 중인 상태. 같은 책임으로 중복 spawn하지 않는다.

waiting

CI, review, 사용자 결정처럼 외부 gate를 기다리는 상태.

done

작업 결과가 PR 또는 branch에 반영된 상태.

closed

close_agent 호출과 worktree cleanup 또는 보존 사유 기록까지 끝난 상태.

abandoned

실패, 중단, 오염 감지. push 금지, agent close, worktree 상태 기록 후 새 owner로 재시작한다.

GPG Signing Rule

새 delivery 규칙

GPG signing 실패나 pinentry timeout이 발생하면 즉시 멈춘다. agent는 --no-gpg-sign이나 commit.gpgsign=false로 진행하지 않는다.

hook 보강

Bash hook은 signing bypass 명령을 차단하고, pre-push는 outgoing commit 중 unsigned commit을 차단한다. 사용자가 GPG cache를 warm-up한 뒤 재개한다.

수정 PR 범위

  1. 문서화 SOT와 Documentation Impact Gate를 workflow memory에 추가한다.
  2. pr-reviewer 관점에 documentation topology와 portable evidence를 추가한다.
  3. delivery owner가 review, reflect, merge, cleanup까지 소유하도록 pipeline을 고친다.
  4. agent lifecycle registry와 close/cleanup 체크리스트를 worktree runbook에 추가한다.
  5. GPG signing bypass 차단과 unsigned commit pre-push gate를 추가한다.
