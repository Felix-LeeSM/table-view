Browser 에서 theme 이 작동하는 원리

CSS variable + selector specificity + data-attribute = 본 프로젝트 theme 시스템

1. CSS variable — 변수 같은 거

일반 CSS 는 값을 하드코딩한다 — color: red;. 변수가 있으면 한 곳에서 정의하고 여러 곳에서 참조 가능. CSS 의 변수 문법:

--my-color: red;
color: var(--my-color);

이름이 반드시 -- 두 줄로 시작해야 한다. 참조할 땐 var(--이름).

2. Cascade — 같은 변수 여러 곳에 정의되면?

같은 변수가 여러 selector 에 정의되면 browser 는 specificity (구체성) 가 높은 selector 의 값을 우선한다.

:root {
  --my-color: red;       /* specificity = (0,1,0) */
}
.special {
  --my-color: blue;      /* specificity = (0,1,0) */
}
#header .special {
  --my-color: green;     /* specificity = (1,1,0) — 가장 specific */
}

specificity 는 보통 3자리 (a,b,c) 로 표현. a=ID 개수, b=클래스/attribute 개수, c=tag 개수. 왼쪽이 클수록 이김.

SelectorSpecificity이김?
:root(0,1,0)red
.special(0,1,0)blue (같은 specificity → 마지막 정의 이김)
#header .special(1,1,0)green ✓

3. data-attribute selector — 본 프로젝트의 핵심 트릭

HTML element 에 data-* attribute 를 박을 수 있다. CSS 는 그걸 selector 로 매칭 가능:

<html data-theme="slate" data-mode="light">
  ...
</html>
[data-theme="slate"] {
  /* html 의 data-theme 이 slate 일 때만 발동 */
  --bg: #fafafa;
}
[data-theme="github"] {
  /* data-theme 이 github 일 때만 발동 */
  --bg: #ffffff;
}

specificity 는 attribute selector 가 (0,1,0) (class 와 동일). [data-theme] 와 [data-mode] 둘 다 쓰면 (0,2,0).

4. 본 프로젝트의 실제 흐름

JavaScript (themeBoot.ts) 가 페이지 로드 시 <html> 에 data-theme 과 data-mode 를 박는다:

// themeBoot.ts
document.documentElement.setAttribute("data-theme", themeId);
document.documentElement.setAttribute("data-mode", "light");

그러면 themes.css 의 selector 들 중 매칭되는 블록의 변수 값이 활성화:

[data-theme="slate"][data-mode="light"] {  /* (0,2,0) */
  --tv-syntax-keyword: #7c3aed;
  ...
}
[data-theme="slate"][data-mode="dark"] {   /* (0,2,0) */
  --tv-syntax-keyword: #c4b5fd;
  ...
}

5. Fallback — 새 테마가 syntax 토큰을 잊으면?

본 결정의 핵심. 새 테마 "tableplus" 가 base 토큰만 정의하고 syntax 토큰을 잊었다고 가정:

[data-theme="tableplus"][data-mode="light"] {  /* (0,2,0) */
  --tv-bg: #fff;
  /* syntax 토큰 정의 X — 잊음 */
}

이 상태에서 color: var(--tv-syntax-keyword) 하면 → tableplus 블록엔 정의 없음. browser 는 specificity 가 더 낮은 다른 selector 에서 찾는다.

그래서 더 낮은 specificity 의 fallback block 을 따로 만들어둠:

:root[data-mode="light"] {  /* (0,1,1) — tableplus 블록 (0,2,0) 보다 낮음 */
  --tv-syntax-keyword: #7c3aed;  /* slate light 값 */
  ...
}

cascade 동작:

변수tableplus 블록 (0,2,0)fallback (0,1,1)발동 값
--tv-bg#fff(미정의)#fff
--tv-syntax-keyword(미정의)#7c3aed#7c3aed (slate fallback)
핵심 — tableplus 가 정의한 토큰만 override, 잊은 토큰은 자동으로 fallback (slate) 발동. 빈 색 / 깨진 highlight 0. 디자이너가 나중에 채우면 그 값이 자동으로 fallback 을 덮음.

7. 결정 — 어느 selector 로 fallback 박을지

(i) :root 단독 — mode 무관 한 세트

specificity (0,1,0). 가장 낮음. 단 하나만 정의 — light fallback 박으면 dark 모드에서도 light syntax 가 발동 → 어두운 bg 에 진한 색 → 대비 깨짐.

✗ light/dark 구분 못함. 권장 안 함.

(ii) :root[data-mode="light"] + :root[data-mode="dark"] — 분리

specificity (0,1,1). 테마 블록 (0,2,0) 보다 낮음 → fallback 으로 작동. 두 selector 분리되어 mode 별 적절 fallback.

✓ 추천. 본 프로젝트의 themeBoot 가 박는 data-mode 와 자연스럽게 매칭.

(iii) :root 에 light + @media (prefers-color-scheme: dark) 로 dark

OS 시스템 dark mode 따라감. 단 본 프로젝트는 themeBoot 가 explicit data-mode 박으므로, user 선택 (e.g. light theme 강제) 과 OS 설정 (dark) 이 충돌하면 OS 가 이김 → 혼란.

✗ 본 프로젝트 user-controlled mode 시스템과 충돌. 권장 안 함.
