---
title: 피드백은 스펙 합의 → TDD 순서로 처리한다
type: lesson
date: 2026-04-27
---

**상황**: 사용자 피드백 12건 중 #1은 "혼자 결정 말고 같이 논의"라 명시했는데, 곧장 e2e 스펙 파일에 12건 모두 테스트 본문/outline부터 작성해 들어갔다.
**원인**: 피드백을 "구현 작업 큐"로 취급해 스펙 합의 단계를 건너뛰었고, 항목별 동작 정의가 모호한 상태에서 테스트가 desired behavior를 추측하게 됐다.
**재발 방지**: 피드백 도착 시 (1) 항목별 상세 스펙을 사용자와 먼저 합의 → (2) 합의된 스펙을 실패하는 테스트로 고정(TDD red, 단위/통합/e2e 무엇이든 항목 성격에 맞는 레이어 선택) → (3) 구현으로 green, 순서 엄수 — 항목 수만큼 이 사이클을 돈다. 단, 구현보다 테스트가 반드시 먼저다.
