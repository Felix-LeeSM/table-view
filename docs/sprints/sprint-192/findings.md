# Sprint 192 — Findings

Date: 2026-05-02. Status: closed.

## 1. Scope 확장 결정

### 1.1 1차 → 2차: data + database 통합

1차 contract 는 schema migration DDL only. 사용자가 진행 중 "실제
record 는 어떻게 한번에 내보낼 수 있지? database 단위 export 도 가능해야
할 것 같아. 무거우니 rust 가 만들어주면 되지 않을까" 로 scope 확장 요청.

**결정**: 별 sprint 분리 (Sprint 196 후보) 가 아니라 본 sprint 안으로
통합. 사용자 의도 — "묶어서 재설계". 진입점 UI 가 결국 같은 헤더 Popover
한 곳이라 entry point 통합과 implementation 분리가 어색.

**결과**: contract size 3~4 일 → 7~9 일 추정 → 실제 1 일 (하루 안에 완료).
큰 design decision 한 번에 묶는 편이 review 단위로 더 자연스러웠음.

### 1.2 Cursor vs LIMIT/OFFSET

1차 design 제안은 LIMIT/OFFSET 일관 fallback 이었으나 사용자 "cursor 로
바로 가자" 결정. PG `DECLARE NO SCROLL CURSOR FOR ... FETCH FORWARD N`
로 정공법.

**Trade-off**:
- (+) 메모리 안전 — server side 가 cursor 로 N row 만 buffer.
- (+) `row_to_json` 으로 PG 모든 native type 을 JSON 으로 표현.
- (–) PG-only — MySQL/SQLite 는 dialect 별 streaming 으로 별도 구현 필요.
  현재 MySQL/SQLite 는 adapter 자체가 Phase 9 placeholder 이므로 무관.

### 1.3 Trait method 추가 vs 회피

1차 design 의 "trait method 회피" (raw SQL 을 command 가 직접 발사) 는
paradigm-leaky. 사용자 "trait 추가로 정공법으로 간다" 결정.

**결정**: `RdbAdapter::stream_table_rows` 신규 trait method (default
`Unsupported`). PostgresAdapter override. MySQL/SQLite Phase 9 합류 시
default 만 override 하면 됨.

## 2. UX 결정

### 2.1 우클릭 → 헤더 Popover

원래 contract AC-192-04 는 schema row context menu 진입점. 사용자 피드백
"우클릭으로 뜨는 건 좀 이상하군 — schema 단위로 export 하도록 되어있나"
로 헤더 Popover 로 이전.

**근거**: (a) 발견성 — 우클릭은 사용자가 시도해봐야 발견. (b) MySQL/SQLite
tree shape 에서 schema row 자체가 hide → 우클릭 진입 불가능. (c) 헤더
Popover 가 single-place entry point 로 일관.

### 2.2 Popover 레이아웃 반복

3 차례 iteration:
1. width `w-72` + 항목별 텍스트 ("Database dump (all schemas)") — 사용자
   "좌우로 너무 뚱뚱". 옵션 모호 ("DDL/DML/전체 어느 것?").
2. width `w-56` + 3 icon (FileText/Rows3/Database) + native title 로 의미
   명시. 사용자 만족.
3. `align="end"` → `align="start"` — 사용자가 trigger 의 왼쪽 edge 정렬
   선호.

**결과**: schema 행 마다 `[Schema] [Data] [Full]` 3 icon 으로 6 액션 모두
한 번에 노출 — 단일 click 으로 dispatch.

### 2.3 ExportButton 일관성 정리

DataGrid toolbar 의 ExportButton 도 schemas Popover 와 결을 맞춰 icon-only
+ align=start. test 영향 없음 (aria-label="Export" 보존).

## 3. Sequence 처리 (사용자 smoke 발견)

### 3.1 nextval default 미생성 문제

사용자 smoke 에서 `"id" bigint NOT NULL DEFAULT nextval('xxx_seq'::regclass)
PRIMARY KEY` 형태로 emit 된 DDL 이 새 PG 에 import 시 `relation
"xxx_seq" does not exist` 로 fail.

**Fix path 비교**:
- (a) BIGSERIAL 정규화 — PG syntactic sugar 가 sequence 자동 생성 + 이름
  규칙 (`<table>_<col>_seq`) 일치. **채택**.
- (b) `CREATE SEQUENCE` 명시 emit — sequence metadata (START/INCREMENT
  등) 보존. schemaStore 확장 필요. **defer**.

### 3.2 setval reset

BIGSERIAL 정규화 후에도 sequence next value 가 1 로 시작 → row 들이
INSERT 된 후 후속 INSERT 가 PK 충돌. dump 끝에:

```sql
SELECT setval(pg_get_serial_sequence('"s"."t"', 'col'),
  COALESCE((SELECT MAX("col") FROM "s"."t"), 1));
```

backend command 에 `ddl_footer: String` 추가. include in {dml, both} 일
때만 setval emit (TS hook 분기).

### 3.3 sequence v1 한계

후속 sprint 가 다뤄야 할 한계:
- `START WITH` / `INCREMENT BY` / `MIN/MAX VALUE` / `CACHE` / `OWNED BY`
  metadata 손실.
- `CREATE SEQUENCE` 명시 emit 형태로 가야 정확한 round-trip.
- `GENERATED BY DEFAULT/ALWAYS AS IDENTITY` (PG 10+) column 미지원 —
  schemaStore 가 `is_identity` flag 을 미반영.

## 4. Design 한계 (deferred risk)

### 4.1 `active_connections` lock-held during dump

`run_schema_dump` 가 dump 끝까지 `state.active_connections.lock()` 을
들고 있음 — 같은 connection 의 다른 IPC 가 await 으로 대기.

**Mitigation 후보**: `ActiveAdapter` 를 `Box<dyn>` → `Arc<dyn>` 으로
변경하면 lock 일찍 풀고 stream 가능. 단 trait object 의 Arc 화는 다른
command 들의 dispatch 도 영향 — 별 sprint refactor.

### 4.2 PG value escape v1

`row_to_json` 결과가 모두 JSON variant 로 들어옴. bytea (`\\x...`),
timestamp (ISO 8601), uuid 모두 String variant 로 PG implicit cast 에
의존. column type 알면 명시 cast 가 robust 하나 호출자가 column_types
도 같이 보내야 함 — v2 후보.

### 4.3 Single-row INSERT per line

multi-row VALUES batch 미적용. 가독성 + diff-friendly 우선. restore
속도 손해 — 큰 dump 에선 multi-row 가 10x 빠를 수 있음. v2 후보.

### 4.4 ddlGenerator MySQL/SQLite path = dead code

MySQL/SQLite adapter 가 Phase 9 placeholder 이므로 `make_adapter` 가
연결을 거절 → schemaStore 에 metadata 도 없어 export 진입점이 비활성.
ddlGenerator 의 dialect="mysql"/"sqlite" path 는 미래용 dead. Phase 9
합류 시점에 즉시 활용 가능.
